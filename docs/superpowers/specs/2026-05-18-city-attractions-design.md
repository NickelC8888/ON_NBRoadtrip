# City Attractions Section — Design Spec
_Date: 2026-05-18_

## Goal

Fetch attractions near each route stop city from the OpenTripMap API and display them in a new accordion section — organized by city — in both TripDetail and LegDetailPage.

---

## Part 1: Fetch Script

**File:** `scripts/fetchCityAttractions.mjs`
**Run:** `node scripts/fetchCityAttractions.mjs`
**Output:** `src/data/openTripMapCityAttractions.js`

### API call per city

Endpoint:
```
GET https://api.opentripmap.com/0.1/en/places/radius
  ?radius=5000
  &lon={lng}
  &lat={lat}
  &kinds=interesting_places
  &rate=2
  &format=json
  &limit=15
  &apikey={VITE_OPENTRIPMAP_API_KEY}
```

- `radius`: 5 km around stop coords
- `rate=2`: minimum rating filter (0–3 scale)
- `limit=15`: max 15 results per city
- API key read from `process.env.VITE_OPENTRIPMAP_API_KEY` (loaded via dotenv)
- 1100 ms delay between requests to respect free-tier rate limit (1 req/sec)

### Response item shape

```json
{
  "xid": "Q12345",
  "name": "Fort Henry",
  "kinds": "historic,architecture,interesting_places",
  "point": { "lon": -76.48, "lat": 44.23 },
  "rate": 2,
  "dist": 320
}
```

### Category mapping

Map the first matching `kinds` segment to a readable label:

```js
function mapCategory(kinds) {
  if (!kinds) return 'Attraction';
  if (kinds.includes('natural'))            return 'Nature';
  if (kinds.includes('historic'))           return 'Historic site';
  if (kinds.includes('museums'))            return 'Museum';
  if (kinds.includes('architecture'))       return 'Architecture';
  if (kinds.includes('cultural'))           return 'Cultural site';
  if (kinds.includes('parks'))              return 'Park';
  if (kinds.includes('religion'))           return 'Religious site';
  if (kinds.includes('amusements'))         return 'Amusement';
  return 'Attraction';
}
```

Within each city's array, filter out items where `name` is empty or `name` matches an existing entry (case-insensitive). Keep first occurrence.

## Part 2: UI Component

### New file: `src/components/roadtrip/CityAttractionsSection.jsx`

**Props:**
```js
{ trip, filterCities }
// trip: full trip object
// filterCities: string[] | null — null = show all cities
```

**Internal structure:**

```jsx
export default function CityAttractionsSection({ trip, filterCities }) {
  const cityData = OPENTRIPMAP_CITY_ATTRACTIONS[trip.id] || {};
  const cities = Object.keys(cityData).filter(city =>
    filterCities === null || filterCities.includes(city)
  );

  if (cities.length === 0) return null;

  return (
    <div className="space-y-3">
      <SectionLabel>Nearby Attractions</SectionLabel>
      <div className="space-y-2">
        {cities.map(city => (
          <CityAccordion key={city} city={city} attractions={cityData[city]} />
        ))}
      </div>
    </div>
  );
}
```

**`CityAccordion`** (internal to same file):
- Collapsible row matching existing `ItineraryDay` accordion styling
- Header: city name + attraction count badge + chevron
- Body: 2-column grid of `AttractionCard`

**`AttractionCard`** (internal to same file):
- Name (bold), category badge (cream pill), distance in km, link to `https://opentripmap.com/en/card/{xid}`
- Same card border/shadow style as other cards: `border border-stone-200 rounded-xl p-3 bg-white shadow-card`
- No dog/kid badges (OpenTripMap data doesn't reliably support this)

**`SectionLabel`** — reuse the one already defined at the bottom of `TripDetail.jsx`. Export it so `CityAttractionsSection` can import it.

## Part 3: Integration

### TripDetail (`src/components/roadtrip/TripDetail.jsx`)

1. Export `SectionLabel` (currently not exported — add `export` keyword)
2. Import `CityAttractionsSection`
3. Add after the "Day-by-Day Itinerary" section, before the filter controls `<div className="bg-white border...">`:

```jsx
<CityAttractionsSection trip={trip} filterCities={null} />
```

### LegDetailPage (`src/components/roadtrip/LegDetailPage.jsx`)

1. Import `CityAttractionsSection` and `OPENTRIPMAP_CITY_ATTRACTIONS`
2. Derive `filterCities` from `day.title`:

```js
const legCities = (() => {
  const cityData = OPENTRIPMAP_CITY_ATTRACTIONS[trip.id] || {};
  const knownCities = Object.keys(cityData);
  const titleParts = day.title
    .split(/→|to|-|\|/i)
    .map(s => s.replace(/\d+\s*km.*$/i, '').trim())
    .filter(Boolean);
  return knownCities.filter(city =>
    titleParts.some(part =>
      city.toLowerCase().includes(part.toLowerCase()) ||
      part.toLowerCase().includes(city.toLowerCase())
    )
  );
})();
```

3. Add at end of `<div className="space-y-6">`, after Tabs:

```jsx
{legCities.length > 0 && (
  <CityAttractionsSection trip={trip} filterCities={legCities} />
)}
```

## Execution Order

The fetch script must run and the generated `src/data/openTripMapCityAttractions.js` must be committed **before** the UI components are implemented. If the file doesn't exist, the `import` in `CityAttractionsSection.jsx` will fail at build time. Task order in the plan must reflect this dependency.

---

## Out of Scope

- Live runtime API fetching
- Dog/kid suitability badges on city attractions
- Caching or re-fetching logic
- Attractions for inactive trips (only fetched for `eastern-circuit` and `new-brunswick-fundy`)
